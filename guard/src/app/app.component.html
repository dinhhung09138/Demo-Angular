Angular routes guard is an interfaces which can the the route whether or not it should allow navigation to a response route

THis make this decision by looking for a 'true' or 'false' return value from a class which implements the given guard interface.

There are five different type of guards

CanActivate

Check to see if user can visit a route

CanActivateChild

Check to see if user can visit a routes child

CanDeactive

Check to see if user can exit a route

Resolve

Perform route data retrieval before route activation

CanLoad

Check to see if user can route to a module that lazy loaded
